1. Authentication

CrymadX uses JWT (JSON Web Tokens) for authentication. Include the token in the Authorization header for all protected endpoints.

Authorization Header

Authorization: Bearer <your_jwt_token>

1.1 User Authentication

POST /auth/register Public

Register a new user account

Request Body
{
  "email": "user@example.com",
  "password": "securePassword123!",
  "fullName": "John Doe",
  "phone": "+1234567890",
  "country": "US",
  "referralCode": "REF123" // optional
}
Response (201 Created)
{
  "success": true,
  "data": {
    "user": {
      "id": "usr_abc123",
      "email": "user@example.com",
      "fullName": "John Doe",
      "status": "pending_verification",

    "user.kyc_verified",
    "deposit.completed",
    "withdrawal.completed",
    "trade.completed",
    "p2p.dispute_opened"
  ],
  "secret": "your_webhook_secret"
}

EventDescription

user.kyc_rejectedUser KYC rejected

withdrawal.requestedWithdrawal requested by user

withdrawal.failedWithdrawal failed
trade.completedTrade executed
order.filledOrder fully filled
order.cancelledOrder cancelled
p2p.order_createdP2P order created
p2p.dispute_openedP2P dispute opened
p2p.dispute_resolvedP2P dispute resolved

All API errors follow a consistent format:

{
  "success": false,
  "error": {
    "code": "INVALID_CREDENTIALS",
    "message": "Invalid email or password",
    "details": {}
  }
}

INVALID_CREDENTIALS401Wrong email or password
